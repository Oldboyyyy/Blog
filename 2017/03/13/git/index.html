    • 使用撤销命令操作，如下：

    git checkout把文件在工作区的修改全部撤销，这里有两种情况，如下：

    1. 文件自动修改以后，还没放到暂存区，使用撤销修改就回到和版本库一模一样的状态。
    2. 另一种，文件修改以后并且也已经放入暂存区，接着又作了修改，撤销修改就回到了添加暂存区后的状态，只是撤销了最近的这次修改，如图示意：

      注意：命令git checkout -- aa.txt中的 --很重要，没有的话，那么命令就变成了创建分支
